## Integration Audit

### Summary

LivingColor supports TVP through generic project-key configuration, but the **codebase does not encode the canonical TVP stack**. Tests and fixtures still reference `tv5monde/tv5mondeplus-front` (GitLab-style) and `develop` branch, not `abecms/tv5mondeplus-front` + `preprod`.

### UI integration state model

`ui/src/app/delivery/project-integrations.tsx` exposes:

| Status | Meaning |
|--------|---------|
| `missing` | No MCP / credentials |
| `configured` | Credentials present, not live-connected (partial) |
| `connected` | Live MCP session |

Automation is blocked when Jira or active VCS is not `connected`.

### GitLab vs GitHub for TVP

- Default VCS provider is GitLab until `vcs: github` is set in project mapping.
- TVP automation tests still hit `/projects/TVP/gitlab-repos`.
- GitLab treated as compatibility path per audit brief — **not validated as primary TVP path**.
